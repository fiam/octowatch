# Security Policy

## Reporting a Vulnerability

Please do not report security vulnerabilities through public GitHub
issues or pull requests.

Instead, email:

- `alberto@garciahierro.com`

Include:

- a clear description of the issue
- steps to reproduce it
- the affected version or commit, if known
- any suggested mitigation or workaround

I will acknowledge receipt as soon as practical and coordinate a fix
before public disclosure when the report is valid.

## Scope

Relevant reports include:

- credential handling
- token storage or disclosure
- unsafe local file handling
- unintended GitHub write actions
- vulnerabilities in release artifacts or packaging

Reports that are purely feature requests or support questions should go
through the normal issue tracker instead.
